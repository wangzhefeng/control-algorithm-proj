# AGENTS.md

本文件定义 Codex 在本仓库协作开发时必须遵循的规范。

## 1. 项目定位

- 项目名称：`control-algorithm-proj`
- 项目目标：用于学习和实践控制算法（以 PID、状态空间控制与耦合系统示例为主）。
- 主要工作区：`src/`
- 非工作区：`docs/`（仅作为参考资料，Codex 不修改该目录内容）。

## 2. 目录与职责

仓库关键目录/文件说明：

- `.venv/`
  - 由 `uv` 管理的 Python 虚拟环境。
- `docs/`
  - 学习文档（论文、书籍、视频等）。
  - 规则：Codex 不处理、不修改该目录文件。
- `src/`
  - 控制算法示例代码（Python / Matlab）。
  - 规则：Codex 优先处理该目录，重点解决可运行性与可读性问题。
- `pyproject.toml`、`uv.lock`
  - 依赖与锁文件。
  - 规则：若 `src/` 代码缺失依赖，使用 `uv add <package>` 安装并同步更新锁文件。
- `.python-version`
  - Python 版本声明。
- `README.md`
  - 项目总说明（功能、结构、运行方式、进展）的唯一维护入口。

## 3. Codex 工作原则

- 先理解再修改：先定位问题和影响范围，再做最小必要改动。
- 以可运行为优先：先保证脚本可运行，再进行结构优化。
- 避免无关改动：不做与当前任务无关的重命名、大规模格式化、目录迁移。
- 不破坏现有工作：不回滚用户已存在但与当前任务无关的改动。

## 4. 依赖与环境规范

### 4.1 依赖管理

- 仅使用 `uv` 管理依赖。
- 新增依赖命令：

```powershell
uv add <package>
```

- 开发依赖命令：

```powershell
uv add --dev <package>
```

- 禁止直接手改 `uv.lock`。

### 4.2 运行方式

统一使用 `uv run` 执行脚本，例如：

```powershell
uv run python src/pid/basic/pid_demo.py
```

## 5. 代码处理规范（src）

- 优先修复：
  - 缺失依赖
  - 语法错误
  - 明显运行时错误（导入失败、路径错误、参数错误）
- 适度重构：
  - 拆分超长函数
  - 消除重复代码
  - 补充必要注释（简洁、解释“为什么”）
- 编码规范：
  - 默认 UTF-8
  - 尽量保持 ASCII 兼容（已有中文内容可保留）

## 6. 验证与交付规范

每次任务完成前，至少执行以下一种验证：

- 导入验证：

```powershell
uv run python -c "import numpy, matplotlib, scipy"
```

- 语法验证：

```powershell
uv run python -m compileall src
```

如果无法验证，必须在结果中明确说明原因与阻塞点。

## 7. README 维护要求

- 项目说明仅维护根目录 `README.md`。
- 不再创建或维护子目录 README（例如 `src/README.md`）。
- 当项目结构、依赖或运行方式发生变化时，统一更新根 `README.md`：
  - 项目简介
  - 目录结构
  - 环境准备
  - 运行示例
  - 当前已知问题与后续计划

## 8. 提交质量要求

Codex 输出结果应包含：

- 修改了哪些文件
- 为什么修改
- 如何验证
- 剩余风险/待办

## 9. 明确禁止项

- 不修改 `docs/` 目录内容。
- 不使用破坏性 Git 命令（如 `git reset --hard`）除非用户明确要求。
- 不在未说明的情况下引入大规模、不必要的结构性改动。

## 10. 建议执行顺序（每次任务）

1. 阅读任务与相关代码（优先 `src/`）。
2. 识别缺失依赖与运行错误。
3. 进行最小改动修复。
4. 使用 `uv run` 完成验证。
5. 仅更新根目录 `README.md` 与必要说明文件。
6. 输出变更说明与后续建议。
